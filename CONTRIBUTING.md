# Contributing

Thanks for your interest in contributing.

## Ground Rules
- Keep pull requests small and focused.
- One change per pull request.
- Follow the project roadmap and MVP scope.
- Security and privacy come first.

## How to Contribute
1. Open an issue describing the change.
2. Fork the repo and create a feature branch.
3. Make the change with clear commit messages.
4. Open a pull request referencing the issue.

## What We Avoid
- Adding new features outside the current phase
- Large refactors without discussion
- Introducing sensitive data handling without explicit approval
